---
title: "Grant Permission"
description: "Grants a direct permission to a user with an optional scope, conditions, and expiry."
api: "POST /admin/users/{user_id}/permissions"
---

<ParamField path="user_id" type="string" required>
  The UUID of the user to grant a permission to.
</ParamField>

<ParamField body="permission_id" type="string" required>
  The UUID of the permission to grant. Must correspond to an existing active permission.
</ParamField>

<ParamField body="granted_reason" type="string" required>
  A business justification for the grant. Required for accountability.

  <Expandable title="requirements">
    - Cannot be empty or whitespace
  </Expandable>
</ParamField>

<ParamField body="expires_at" type="string" required>
  Expiry timestamp in ISO 8601 format. All direct permission grants are temporary — an expiry is always required.

  <Expandable title="requirements">
    - Must be in the future
    - Enforced server-side: minimum 5 minutes, maximum 90 days from now
  </Expandable>
</ParamField>

<ParamField body="scope" type="string">
  The scope of the grant. Accepted values are `"own"` and `"all"`. Defaults to `"own"` when omitted or unrecognised. Whether a given scope is permitted depends on the permission's `scope_policy`.
</ParamField>

<ParamField body="conditions" type="object">
  An optional JSON object expressing additional constraints on the grant (e.g. rate limits or field-level restrictions). Stored as-is and evaluated at permission-check time. Defaults to `{}` when omitted.
</ParamField>

<Callout icon="key" color="#FFC107">
  Requires the **`rbac:grant_user_permission`** permission.
</Callout>

## Behaviour

Grants a direct permission to the target user. Direct grants are independent of the user's role — they allow fine-grained, time-limited access to a single permission without changing the user's role assignment.

**All grants are temporary.** `expires_at` is always required. Once expired, the grant is no longer active and the user loses the permission.

**Privilege escalation is prevented.** The granting user must themselves hold the permission being granted. Attempting to grant a permission you do not hold will be rejected with a `403`.

**Duplicate grants are rejected.** If the user already holds an active (non-expired) grant for the same permission, the request is rejected with a `409`. Revoke the existing grant first using [`DELETE /admin/users/{user_id}/permissions/{grant_id}`](/api-reference/admin/user-permissions/revoke-user-permission) before re-granting.

**Scope validation** is enforced by the permission's `scope_policy`. Attempting to grant a scope that the permission does not allow returns a `422`.

<RequestExample>
```json Example
{
  "permission_id": "03a1f2b3-4c5d-6e7f-8a9b-0c1d2e3f4a5b",
  "granted_reason": "Temporary access to manage own product listings during campaign.",
  "expires_at": "2024-09-01T00:00:00Z",
  "scope": "own",
  "conditions": {}
}
```
</RequestExample>

<ResponseExample>
```json 201
{
  "id": "04b2c3d4-5e6f-7a8b-9c0d-1e2f3a4b5c6d",
  "canonical_name": "product:manage",
  "name": "Manage Products",
  "resource_type": "product",
  "scope": "own",
  "conditions": {},
  "expires_at": "2024-09-01T00:00:00Z",
  "granted_at": "2024-06-01T10:00:00Z",
  "granted_reason": "Temporary access to manage own product listings during campaign."
}
```

```json 409
{
  "code": "permission_already_granted",
  "message": "permission already granted to this user"
}
```

```json 422
{
  "code": "permission_id_required",
  "message": "permission_id is required"
}
```

```json 403
{
  "code": "privilege_escalation",
  "message": "granter does not hold this permission"
}
```

```json 401
{
  "code": "unauthorized",
  "message": "missing or invalid access token"
}
```
</ResponseExample>

## Responses

<AccordionGroup>
  <Accordion title="201 — Created">
    The permission was successfully granted. Returns the full grant record.

    ```json
    {
      "id": "04b2c3d4-5e6f-7a8b-9c0d-1e2f3a4b5c6d",
      "canonical_name": "product:manage",
      "name": "Manage Products",
      "resource_type": "product",
      "scope": "own",
      "conditions": {},
      "expires_at": "2024-09-01T00:00:00Z",
      "granted_at": "2024-06-01T10:00:00Z",
      "granted_reason": "Temporary access to manage own product listings during campaign."
    }
    ```
  </Accordion>

  <Accordion title="401 — Unauthorized">
    The request is missing a valid access token.

    ```json
    {
      "code": "unauthorized",
      "message": "missing or invalid access token"
    }
    ```
  </Accordion>

  <Accordion title="403 — Forbidden">
    Returned in two cases: the authenticated user does not hold the `rbac:grant_user_permission` permission, or the granter does not themselves hold the permission they are attempting to grant (privilege escalation).

    ```json
    {
      "code": "forbidden",
      "message": "forbidden"
    }
    ```

    ```json
    {
      "code": "privilege_escalation",
      "message": "granter does not hold this permission"
    }
    ```
  </Accordion>

  <Accordion title="404 — Not Found">
    The `user_id` is not a valid UUID or does not correspond to an existing user.

    ```json
    {
      "code": "user_not_found",
      "message": "user not found"
    }
    ```
  </Accordion>

  <Accordion title="409 — Conflict">
    The user already holds an active (non-expired) direct grant for this permission. Revoke the existing grant before re-granting.

    ```json
    {
      "code": "permission_already_granted",
      "message": "permission already granted to this user"
    }
    ```
  </Accordion>

  <Accordion title="422 — Validation Error">
    One or more required fields are missing, invalid, or violate a business rule.

    ```json
    {
      "code": "permission_id_required",
      "message": "permission_id is required"
    }
    ```

    ```json
    {
      "code": "granted_reason_required",
      "message": "granted_reason is required"
    }
    ```

    ```json
    {
      "code": "expires_at_required",
      "message": "expires_at is required"
    }
    ```

    ```json
    {
      "code": "expires_at_in_past",
      "message": "expires_at must be in the future"
    }
    ```

    ```json
    {
      "code": "scope_not_allowed",
      "message": "scope is not permitted for this permission"
    }
    ```

    ```json
    {
      "code": "permission_not_found",
      "message": "permission not found"
    }
    ```
  </Accordion>

  <Accordion title="500 — Internal Server Error">
    An unexpected error occurred on the server.

    ```json
    {
      "code": "internal_error",
      "message": "internal server error"
    }
    ```
  </Accordion>
</AccordionGroup>
