---
title: "Get User Role"
description: "Returns the active role assignment for a given user."
api: "GET /admin/users/{user_id}/role"
---

<ParamField path="user_id" type="string" required>
  The UUID of the user whose role assignment you want to retrieve.
</ParamField>

<Callout icon="key" color="#FFC107">
  Requires the **`rbac:read`** permission.
</Callout>

## Behaviour

Returns the user's current active role assignment. A role is considered active when it has not expired — permanent grants have no expiry, while temporary grants are active only until their expiry time passes.

If the user has no active role assignment, a `404` is returned. This includes cases where a temporary role has expired but not yet been cleaned up.

<ResponseExample>
```json 200
{
  "user_id": "018e4c3a-1b2d-7f4e-9c0a-2d3e4f5a6b7c",
  "role_id": "029f5d4b-2c3e-8a5f-0d1b-3e4f5a6b7c8d",
  "role_name": "Store Manager",
  "is_owner_role": false,
  "granted_reason": "Promoted to store manager after onboarding.",
  "granted_at": "2024-06-01T10:00:00Z",
  "expires_at": null
}
```

```json 404
{
  "code": "user_role_not_found",
  "message": "user has no active role assignment"
}
```

```json 401
{
  "code": "unauthorized",
  "message": "missing or invalid access token"
}
```

```json 403
{
  "code": "forbidden",
  "message": "forbidden"
}
```
</ResponseExample>

## Responses

<AccordionGroup>
  <Accordion title="200 — OK">
    The user's active role assignment. `expires_at` is `null` for permanent grants and an ISO 8601 timestamp for temporary ones.

    ```json
    {
      "user_id": "018e4c3a-1b2d-7f4e-9c0a-2d3e4f5a6b7c",
      "role_id": "029f5d4b-2c3e-8a5f-0d1b-3e4f5a6b7c8d",
      "role_name": "Store Manager",
      "is_owner_role": false,
      "granted_reason": "Promoted to store manager after onboarding.",
      "granted_at": "2024-06-01T10:00:00Z",
      "expires_at": null
    }
    ```
  </Accordion>

  <Accordion title="401 — Unauthorized">
    The request is missing a valid access token.

    ```json
    {
      "code": "unauthorized",
      "message": "missing or invalid access token"
    }
    ```
  </Accordion>

  <Accordion title="403 — Forbidden">
    The authenticated user does not hold the `rbac:read` permission.

    ```json
    {
      "code": "forbidden",
      "message": "forbidden"
    }
    ```
  </Accordion>

  <Accordion title="404 — Not Found">
    The user has no active role assignment. This is returned both when the user has never been assigned a role and when their temporary assignment has expired.

    ```json
    {
      "code": "user_role_not_found",
      "message": "user has no active role assignment"
    }
    ```
  </Accordion>

  <Accordion title="500 — Internal Server Error">
    An unexpected error occurred on the server.

    ```json
    {
      "code": "internal_error",
      "message": "internal server error"
    }
    ```
  </Accordion>
</AccordionGroup>
