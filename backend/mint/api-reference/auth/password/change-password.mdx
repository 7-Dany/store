---
title: "Change Password"
description: "Changes the password for the currently signed-in account."
api: "POST http://localhost:8080/api/v1/auth/change-password"
---

<ParamField body="old_password" type="string" required>
  The account's current password.
</ParamField>

<ParamField body="new_password" type="string" required>
  The new password to set.

  <Expandable title="requirements">
    - 8–72 characters
    - At least one uppercase letter
    - At least one lowercase letter
    - At least one digit
    - At least one symbol
    - Must differ from the current password
  </Expandable>
</ParamField>

## Behaviour

**All sessions revoked** — A successful change signs out every active session across all devices. The current access token is immediately invalidated, so the client must log in again to obtain a fresh one.

**Wrong password attempts** — Submitting an incorrect current password increments a per-account counter. After 5 consecutive wrong attempts the endpoint returns `429 too_many_requests` and directs the user to the forgot-password flow. A successful password change resets the counter.

## Rate Limiting

**5 requests per 15 minutes per IP address**.

Clients that exceed this limit will receive a `429 Too Many Requests` response with a `Retry-After` header indicating when they may retry.

<RequestExample>
```json Example
{
  "old_password": "P@ssw0rd1!",
  "new_password": "NewP@ssw0rd2!"
}
```
</RequestExample>

<ResponseExample>
```json 200
{
  "message": "password changed successfully"
}
```

```json 401
{
  "code": "invalid_credentials",
  "message": "current password is incorrect"
}
```

```json 422
{
  "code": "validation_error",
  "message": "new password must differ from current password"
}
```

```json 429
{
  "code": "too_many_requests",
  "message": "too many requests — please slow down"
}
```
</ResponseExample>

## Responses

<AccordionGroup>
  <Accordion title="200 — OK">
    The password was changed successfully. All active sessions have been signed out and the current access token is now invalid. Log in again to continue.

    ```json
    {
      "message": "password changed successfully"
    }
    ```
  </Accordion>

  <Accordion title="401 — Unauthorized">
    The current password supplied in `old_password` is incorrect.

    ```json
    {
      "code": "invalid_credentials",
      "message": "current password is incorrect"
    }
    ```
  </Accordion>

  <Accordion title="422 — Validation Error">
    Returned for several distinct cases — check the `message` field for details.

    **Same password** — The new password is identical to the current one.

    ```json
    {
      "code": "validation_error",
      "message": "new password must differ from current password"
    }
    ```

    **Weak password** — The new password does not meet strength requirements.

    ```json
    {
      "code": "validation_error",
      "message": "password must be at least 8 characters"
    }
    ```

    **Missing field** — A required field was not provided.

    ```json
    {
      "code": "validation_error",
      "message": "old_password is required"
    }
    ```
  </Accordion>

  <Accordion title="429 — Too Many Attempts">
    Returned for two distinct scenarios. Check the `Retry-After` response header in both cases.

    **Wrong password threshold** — The current password has been entered incorrectly 5 times in a row. Use `POST /forgot-password` to regain access.

    ```json
    {
      "code": "too_many_requests",
      "message": "too many incorrect password attempts — use forgot-password to reset your password"
    }
    ```

    **IP rate limit** — The IP address has exceeded 5 requests per 15 minutes.

    ```json
    {
      "code": "too_many_requests",
      "message": "too many requests — please slow down"
    }
    ```
  </Accordion>
</AccordionGroup>
