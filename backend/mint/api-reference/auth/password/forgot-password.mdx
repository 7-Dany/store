---
title: "Forgot Password"
authMethod: "none"
description: "Sends a one-time password reset code to the given email address."
api: "POST /auth/password/reset"
---

<ParamField body="email" type="string" required>
  The email address the account was registered with.

  <Expandable title="constraints">
    - Must be a valid email address
    - Maximum 254 characters
  </Expandable>
</ParamField>

## Behaviour

**Anti-enumeration** — This endpoint always returns `202 Accepted` with the same body regardless of whether the email belongs to a registered, verified, or unknown account. No information about account existence is ever revealed.

**Cooldown** — If a reset code was issued within the last 60 seconds, a second request is silently suppressed. No new code is sent and the response is still `202`.

## Rate Limiting

**3 requests per 10 minutes per IP address**.

Clients that exceed this limit will receive a `429 Too Many Requests` response with a `Retry-After` header indicating when they may retry.

<RequestExample>
```json Example
{
  "email": "alice@example.com"
}
```
</RequestExample>

<ResponseExample>
```json 202
{
  "message": "if that email is registered and verified, a password reset code has been sent"

```

## Responses

<AccordionGroup>
  <Accordion title="202 — Accepted">
    The request was accepted. If the email belongs to a verified account, a 6-digit reset code has been sent to that address. The response is identical whether or not the email exists.

    ```json
    {
      "message": "if that email is registered and verified, a password reset code has been sent"
    }
    ```
  </Accordion>

  <Accordion title="400 — Bad Request">
    The request body is not valid JSON.

    ```json
    {
      "code": "bad_request",
      "message": "malformed JSON body"
    }
    ```
  </Accordion>

  <Accordion title="422 — Validation Error">
    The email field is missing, empty, or not a valid email address.

    ```json
    {
      "code": "validation_error",
      "message": "email is required"
    }
    ```
  </Accordion>

  <Accordion title="429 — Rate Limited">
    The IP address has exceeded 3 requests per 10 minutes. Check the `Retry-After` response header for how many seconds to wait before retrying.

    ```json
    {
      "code": "too_many_requests",
      "message": "too many requests — please slow down"
    }
    ```
  </Accordion>
</AccordionGroup>
