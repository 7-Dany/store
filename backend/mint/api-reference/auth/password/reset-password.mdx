---
title: "Reset Password"
authMethod: "none"
description: "Completes a password reset using the grant token issued by Verify Reset Code."
api: "POST http://localhost:8080/api/v1/auth/reset-password"
---

<ParamField body="reset_token" type="string" required>
  The grant token returned by [`POST /verify-reset-code`](/api-reference/auth/password/verify-reset-code). Valid for 10 minutes and single-use.
</ParamField>

<ParamField body="new_password" type="string" required>
  The new password to set on the account.

  <Expandable title="requirements">
    - 8–72 characters
    - At least one uppercase letter
    - At least one lowercase letter
    - At least one digit
    - At least one symbol
    - Must differ from the current password
  </Expandable>
</ParamField>

## Behaviour

**All sessions revoked** — A successful reset signs out every active session across all devices. Any previously issued access tokens are immediately invalidated.

**Single-use grant token** — The grant token is deleted the moment this endpoint receives it, whether the request succeeds or fails due to an expired or missing token. It cannot be reused.

**Grant token expiry** — The grant token expires 10 minutes after it was issued by [`POST /verify-reset-code`](/api-reference/auth/password/verify-reset-code). An expired or unknown token returns `422`.

## Rate Limiting

**5 requests per 10 minutes per IP address**.

Clients that exceed this limit will receive a `429 Too Many Requests` response with a `Retry-After` header indicating when they may retry.

<RequestExample>
```json Example
{
  "reset_token": "3f2a1b4c-...",
  "new_password": "NewP@ssw0rd1!"
}
```
</RequestExample>

<ResponseExample>
```json 200
{
  "message": "password reset successfully"
}
```

```json 400
{
  "code": "bad_request",
  "message": "malformed JSON body"
}
```

```json 422
{
  "code": "validation_error",
  "message": "invalid or expired reset token"
}
```

```json 429
{
  "code": "too_many_requests",
  "message": "too many requests — please slow down"
}
```
</ResponseExample>

## Responses

<AccordionGroup>
  <Accordion title="200 — OK">
    The password was changed successfully. All active sessions have been signed out and all previously issued access tokens are now invalid. The user must log in again.

    ```json
    {
      "message": "password reset successfully"
    }
    ```
  </Accordion>

  <Accordion title="400 — Bad Request">
    The request body is not valid JSON.

    ```json
    {
      "code": "bad_request",
      "message": "malformed JSON body"
    }
    ```
  </Accordion>

  <Accordion title="422 — Validation Error">
    Returned for several distinct cases — check the `message` field for details.

    **Invalid or expired grant token** — The token was not found, has already been used, or has expired.

    ```json
    {
      "code": "validation_error",
      "message": "invalid or expired reset token"
    }
    ```

    **Same password** — The new password is identical to the current one.

    ```json
    {
      "code": "validation_error",
      "message": "new password must differ from current password"
    }
    ```

    **Weak password** — The new password does not meet strength requirements.

    ```json
    {
      "code": "validation_error",
      "message": "password must be at least 8 characters"
    }
    ```
  </Accordion>

  <Accordion title="429 — Rate Limited">
    The IP address has exceeded 5 requests per 10 minutes. Check the `Retry-After` response header for how many seconds to wait before retrying.

    ```json
    {
      "code": "too_many_requests",
      "message": "too many requests — please slow down"
    }
    ```
  </Accordion>
</AccordionGroup>
