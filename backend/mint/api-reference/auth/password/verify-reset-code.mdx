title: "Verify Reset Code"
description: "Validates a password-reset OTP and returns a short-lived grant token to authorize the password change."
api: "POST http://localhost:8080/api/v1/auth/verify-reset-code"
---

<ParamField body="email" type="string" required>
  The email address the account was registered with.

  <Expandable title="constraints">
    - Must be a valid email address
    - Maximum 254 characters
  </Expandable>
</ParamField>

<ParamField body="code" type="string" required>
  The 6-digit reset code sent to the email address by Forgot Password.

  <Expandable title="constraints">
    - Exactly 6 digits
    - Digits only — no letters or symbols
  </Expandable>
</ParamField>

## Behaviour

**Code is not consumed** — This endpoint validates the OTP without consuming it. The code remains valid and must be presented again (via the grant token) to `POST /reset-password`. This allows the frontend to confirm the code on screen 2 before asking the user to choose a new password on screen 3.

**Grant token** — On success a short-lived grant token is returned. It encodes the verified email and code and is stored server-side with a 10-minute TTL. Pass it directly to `POST /reset-password` — it is single-use and cannot be reused after a successful password reset.

**Attempt tracking** — A wrong code still increments the per-token attempt counter. Too many wrong attempts lock the code and return `429`, at which point the user must restart from `POST /forgot-password`.

## Rate Limiting

**5 requests per 10 minutes per IP address**.

Clients that exceed this limit will receive a `429 Too Many Requests` response with a `Retry-After` header indicating when they may retry.

## Responses

<AccordionGroup>
  <Accordion title="200 — OK">
    The code is valid. A grant token is returned with a 10-minute TTL. Pass `reset_token` to `POST /reset-password` to complete the password change.

    ```json
    {
      "reset_token": "3f2a1b4c-...",
      "expires_in": 600
    }
    ```
  </Accordion>

  <Accordion title="410 — Gone">
    The reset code has expired. Request a new code via `POST /forgot-password`.

    ```json
    {
      "code": "token_expired",
      "message": "reset code has expired"
    }
    ```
  </Accordion>

  <Accordion title="422 — Validation Error">
    Returned for several distinct cases — check the `message` field for details.

    **Invalid code** — The submitted code does not match, has already been used, or no active reset was requested for this email.

    ```json
    {
      "code": "validation_error",
      "message": "invalid reset code"
    }
    ```

    **Missing or malformed field** — A required field is absent or fails format validation.

    ```json
    {
      "code": "validation_error",
      "message": "email is required"
    }
    ```
  </Accordion>

  <Accordion title="429 — Too Many Attempts">
    Returned for two distinct scenarios. Check the `Retry-After` response header in both cases.

    **Wrong code threshold** — The code has been entered incorrectly too many times and is now locked. Request a fresh code via `POST /forgot-password`.

    ```json
    {
      "code": "too_many_attempts",
      "message": "too many incorrect attempts"
    }
    ```

    **IP rate limit** — The IP address has exceeded 5 requests per 10 minutes.

    ```json
    {
      "code": "rate_limited",
      "message": "too many requests"
    }
    ```
  </Accordion>
</AccordionGroup>
