---
title: "Block Details"
description: "Look up compact metadata for a Bitcoin block by block hash."
api: "GET /bitcoin/block/{hash}"
---

<ParamField path="hash" type="string" required>
  The block hash to look up. Must be a 64-character hex string. Upper and
  lower case are both accepted — the value is normalised to lowercase before lookup.
</ParamField>

## Behaviour

Returns compact metadata for a single block. This response is suitable for a
frontend details panel opened from a `new_block` event or a block-history view.

Fields returned:

| Field | Type | Notes |
|---|---|---|
| `hash` | string | Canonical block hash |
| `confirmations` | integer | `-1` if the block is not on the active chain |
| `height` | integer | Block height |
| `version` | integer | Block version |
| `merkle_root` | string | Merkle root hash |
| `time` | integer | Block timestamp (Unix seconds) |
| `median_time` | integer | Median time past (Unix seconds) |
| `nonce` | integer | Mining nonce |
| `bits` | string | Encoded target threshold |
| `difficulty` | number | Difficulty at this block |
| `chainwork` | string | Chainwork hex |
| `tx_count` | integer | Number of transactions in the block |
| `previous_block_hash` | string | Omitted for genesis |
| `next_block_hash` | string | Omitted for current tip |

## Rate Limiting

**20 requests per 1 minute per IP address**.

The rate limiter runs **before** JWT authentication. Requests that exceed the
limit are rejected with `429` before token validation occurs.

Clients that exceed this limit will receive a `429 Too Many Requests` response
with a `Retry-After` header indicating when they may retry.
