---
title: "Create Tracked Tx"
description: "Create one durable Bitcoin transaction-tracking row for an explicit txid."
api: "POST http://localhost:8080/api/v1/bitcoin/tx"
---

<ParamField body="network" type="string" required>
  Must match the server's configured Bitcoin network.
</ParamField>

<ParamField body="txid" type="string" required>
  A 64-character hex transaction ID. Uppercase input is normalised to lowercase.
</ParamField>

<ParamField body="address" type="string">
  Optional Bitcoin address to associate with the tracked row. Must be valid for
  the configured network when supplied.
</ParamField>

## Behaviour

Creates one saved tracked-transaction record for an explicit txid. Before the
record is created, the API resolves the current status of the transaction so the
saved row starts in `confirmed`, `mempool`, `not_found`, `conflicting`, or
`abandoned` immediately.

This endpoint is for explicit txid tracking. Watch-discovered rows are created
automatically when watched addresses match transactions and cannot be created
manually here.

<RequestExample>
```json Example
{
  "network": "testnet4",
  "txid": "59a32e91550ce5e1fd33cdd4f2ad8ac33efa3fb9cfac061893d5b73a210a463b",
  "address": "tb1qfc74wvs6wnz3p2twgza26vukqct4emt2v47xwq"
}
```
</RequestExample>
