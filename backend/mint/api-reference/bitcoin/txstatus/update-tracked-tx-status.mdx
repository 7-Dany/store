---
title: "Update Tracked Tx"
description: "Refresh one explicit Bitcoin txid-tracking row and optionally replace its associated address."
api: "PUT http://localhost:8080/api/v1/bitcoin/tx/{id}"
---

<ParamField path="id" type="integer" required>
  The durable row ID to update.
</ParamField>

<ParamField body="address" type="string">
  Optional replacement Bitcoin address. Must be valid for the configured network.
  Omit this field to clear the associated address.
</ParamField>

## Behaviour

Refreshes one explicit `tracking_mode = "txid"` row using its existing txid and
optionally replaces its associated address.

Watch-managed rows (`tracking_mode = "watch"`) are immutable here because they
are updated automatically when watched addresses match transactions.

<RequestExample>
```json Example
{
  "address": "tb1qfc74wvs6wnz3p2twgza26vukqct4emt2v47xwq"
}
```
</RequestExample>

<ResponseExample>
```json 200
{
  "id": 7,
  "network": "testnet4",
  "tracking_mode": "txid",
  "address": "tb1qfc74wvs6wnz3p2twgza26vukqct4emt2v47xwq",
  "txid": "59a32e91550ce5e1fd33cdd4f2ad8ac33efa3fb9cfac061893d5b73a210a463b",
  "status": "mempool",
  "confirmations": 0,
  "amount_sat": 0,
  "fee_rate_sat_vbyte": 0,
  "first_seen_at": "2026-03-30T16:10:00Z",
  "last_seen_at": "2026-03-30T16:20:00Z",
  "created_at": "2026-03-30T16:10:00Z",
  "updated_at": "2026-03-30T16:20:00Z"
}
```

```json 409
{
  "code": "watch_managed_record",
  "message": "watch-managed transaction status records cannot be updated here"
}
```
</ResponseExample>
