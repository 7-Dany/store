---
title: "Update Watch"
description: "Update one Bitcoin watch resource without changing its watch type."
api: "PUT http://localhost:8080/api/v1/bitcoin/watch/{id}"
---

<ParamField path="id" type="integer" required>
  The durable watch resource ID.
</ParamField>

<ParamField body="watch_type" type="string" required>
  Must match the watch resource's existing type. Cross-type mutation is rejected.
</ParamField>

<ParamField body="address" type="string">
  Required when `watch_type = "address"`. Must be omitted when `watch_type = "transaction"`.
</ParamField>

<ParamField body="txid" type="string">
  Required when `watch_type = "transaction"`. Must be omitted when `watch_type = "address"`.
</ParamField>

## Behaviour

Updates one existing watch resource owned by the authenticated user.

`watch_type` is immutable after creation. The request must keep the same type and
replace only the target within that type.

## Rate Limiting

**10 requests per 1 minute per IP address**.

<RequestExample>
```json Example
{
  "watch_type": "transaction",
  "txid": "59a32e91550ce5e1fd33cdd4f2ad8ac33efa3fb9cfac061893d5b73a210a463b"
}
```
</RequestExample>

<ResponseExample>
```json 200
{
  "id": 11,
  "network": "testnet4",
  "watch_type": "transaction",
  "txid": "59a32e91550ce5e1fd33cdd4f2ad8ac33efa3fb9cfac061893d5b73a210a463b",
  "status": "active",
  "created_at": "2026-03-31T00:30:00Z",
  "updated_at": "2026-03-31T00:45:00Z"
}
```

```json 409
{
  "code": "watch_exists",
  "message": "watch already exists"
}
```

```json 429
{
  "code": "too_many_requests",
  "message": "too many requests — please slow down"
}
```
</ResponseExample>
