## Base URL

```
http://localhost:8080/api/v1
```

## Authentication

Protected endpoints require a valid JWT access token in the `Authorization` header:

```
Authorization: Bearer <access_token>
```

Access tokens are short-lived (15 minutes). Use [`POST /auth/refresh`](/api-reference/auth/session/refresh) to obtain a new pair before expiry. See the [Session Lifecycle](/guides/auth/session-lifecycle) guide for the full token rotation flow.

## Content type

All requests that include a body must set:

```
Content-Type: application/json
```

Endpoints that require a body return `415 Unsupported Media Type` when this header is missing or incorrect.
