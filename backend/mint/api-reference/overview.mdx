## Base URL

```
http://localhost:8080
```

All requests must use `Content-Type: application/json`.

---

## Errors

All error responses share the same shape:

```json
{
  "code": "error_code",
  "message": "human readable description"
}
```

| Status | Meaning                                      |
| ------ | -------------------------------------------- |
| 400    | Bad request                                  |
| 401    | Unauthenticated                              |
| 403    | Forbidden                                    |
| 404    | Not found                                    |
| 409    | Conflict (e.g. email already taken)          |
| 413    | Request body too large                       |
| 415    | Wrong Content-Type                           |
| 422    | Validation error                             |
| 429    | Rate limited — check `Retry-After` header    |

## Permissions Reference

Admin endpoints that require a specific permission list it in their documentation. Use this as a reference when assigning permissions to roles via [`POST /admin/rbac/roles/{id}/permissions`](/api-reference/rbac/roles/add-role-permission).

The `canonical_name` is the exact value used in permission checks. The `scope_policy` and `access_types` come from each permission's [`capabilities`](/api-reference/rbac/permissions/list-permissions) object and determine which `scope` and `access_type` values are valid for grants:

- **`scope_policy: none`** — scope has no meaning for this permission; hide the scope field in your UI. The service normalises the stored value to `"all"` automatically.
- **`scope_policy: any`** — caller chooses `"own"` or `"all"`; show a scope selector.
- **`access_types`** — only the listed values are accepted. `direct` and `denied` are always present. `request` and `conditional` appear only when explicitly enabled. Passing anything outside this list returns `422 access_type_not_allowed`.
- **`request` at runtime** — when a user calls an endpoint guarded by a `request`-type permission, the action is not executed immediately. The API returns `202` with `{ "code": "approval_required", "request_id": "..." }`. The action only executes once an approver approves the submitted request.
