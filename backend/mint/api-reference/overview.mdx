## Base URL

```
http://localhost:8080
```

All requests must use `Content-Type: application/json`.

---

## Errors

All error responses share the same shape:

```json
{
  "code": "error_code",
  "message": "human readable description"
}
```

| Status | Meaning                                      |
| ------ | -------------------------------------------- |
| 400    | Bad request                                  |
| 401    | Unauthenticated                              |
| 403    | Forbidden                                    |
| 404    | Not found                                    |
| 409    | Conflict (e.g. email already taken)          |
| 413    | Request body too large                       |
| 415    | Wrong Content-Type                           |
| 422    | Validation error                             |
| 429    | Rate limited — check `Retry-After` header    |

## Permissions Reference

Admin endpoints that require a specific permission list it in their documentation. Use this table as a reference when assigning permissions to roles.

The `canonical_name` column is the exact value used in permission checks — use it when granting permissions via the API.

| Canonical name               | Resource type | Description                                          |
| ---------------------------- | ------------- | ---------------------------------------------------- |
| `rbac:read`                  | rbac          | View roles, permissions, and user role assignments   |
| `rbac:manage`                | rbac          | Create and modify roles and role assignments         |
| `rbac:grant_user_permission` | rbac          | Grant or revoke direct permissions on a user         |
| `job_queue:read`             | job_queue     | View job queues and their current status             |
| `job_queue:manage`           | job_queue     | Pause, resume, and drain job queues                  |
| `job_queue:configure`        | job_queue     | Change queue configuration and worker settings       |
| `user:read`                  | user          | View user accounts and profile data                  |
| `user:manage`                | user          | Create and update user accounts                      |
| `user:lock`                  | user          | Lock and unlock user accounts                        |
| `request:read`               | request       | View pending and historical approval requests        |
| `request:manage`             | request       | Submit and cancel approval requests                  |
| `request:approve`            | request       | Approve or reject pending requests                   |
| `product:manage`             | product       | Create, update, and delete products                  |

<Info>
  The owner role bypasses all permission checks and has unrestricted access to every endpoint.
</Info>
