---
title: "Update Profile"
description: "Updates the authenticated user's display name and/or avatar URL."
api: "PATCH /profile/me"
---

<ParamField body="display_name" type="string">
  The name displayed on the user's profile.

  <Expandable title="requirements">
    - Must not be empty or whitespace-only
    - Maximum 100 characters
    - Must not contain ASCII control characters
  </Expandable>
</ParamField>

<ParamField body="avatar_url" type="string">
  A publicly accessible URL pointing to the user's avatar image.

  <Expandable title="requirements">
    - Must be a valid absolute URL
    - Must use an allowed scheme (e.g. `https`)
    - Maximum 2048 bytes
    - Cannot be an empty string
  </Expandable>
</ParamField>

## Behaviour

**Partial updates** — Both fields are optional. Omitting a field (or sending `null`) leaves it unchanged. At least one field must be provided with a non-null value.

**No no-op detection** — Submitting the same values already stored is accepted and returns `200 OK`.

## Rate Limiting

**10 requests per 1 minute per IP address**.

Clients that exceed this limit will receive a `429 Too Many Requests` response with a `Retry-After` header indicating when they may retry.

## Responses

<AccordionGroup>
  <Accordion title="200 — OK">
    The profile was updated successfully. Omitted fields are left unchanged.

    ```json
    {
      "message": "profile updated successfully"
    }
    ```
  </Accordion>

  <Accordion title="400 — Bad Request">
    The request body could not be parsed as valid JSON. Common causes: malformed JSON syntax (`{bad json`) or a string field containing a literal ASCII control character (e.g. `\u0001`).

    ```json
    {
      "code": "bad_request",
      "message": "malformed JSON body"
    }
    ```
  </Accordion>

  <Accordion title="401 — Unauthorized">

    ```json
    {
      "code": "unauthorized",
      "message": "missing user id in context"
    }
    ```
  </Accordion>

  <Accordion title="413 — Payload Too Large">
    The request body exceeds the allowed limit.

    ```json
    {
      "message": "request body too large"
    }
    ```
  </Accordion>
