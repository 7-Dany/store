---
title: "Cancel Deletion"
description: "Cancel a pending account deletion, restoring the account to active status."
api: "DELETE /profile/me/deletion"
---

This endpoint takes no request body.

## Behaviour

Cancels a pending deletion for an account that was previously scheduled via
[`DELETE /profile/me`](/api-reference/profile/deletion/delete-account).
The account is immediately restored to full active status and the
`scheduled_deletion_at` field is no longer returned in
[`GET /profile/me`](/api-reference/profile/account/me) responses.

The endpoint is idempotent in intent — calling it when no deletion is pending
returns `409 not_pending_deletion` rather than silently succeeding, so callers
can distinguish a genuine cancel from a no-op.

## Rate Limiting

**5 requests per 10 minutes per user**.

Clients that exceed this limit will receive a `429 Too Many Requests` response
with a `Retry-After` header indicating when they may retry.

<ResponseExample>
```json 200
{
  "message": "account deletion cancelled"
}
```

```json 409
{
  "code": "not_pending_deletion",
  "message": "account is not scheduled for deletion"
}
```

```json 429
{
  "code": "too_many_requests",
  "message": "too many requests — please slow down"
}
```
</ResponseExample>

## Responses

<AccordionGroup>
  <Accordion title="200 — OK">
    The pending deletion has been cancelled. The account is fully active again.

    ```json
    {
      "message": "account deletion cancelled"
    }
    ```
  </Accordion>

  <Accordion title="401 — Unauthorized">
    No valid access token was provided.

    ```json
    {
      "code": "unauthorized",
      "message": "missing user id in context"
    }
    ```
  </Accordion>

  <Accordion title="409 — Conflict">
    The account does not have a pending deletion. There is nothing to cancel.

    ```json
    {
      "code": "not_pending_deletion",
      "message": "account is not scheduled for deletion"
    }
    ```
  </Accordion>

  <Accordion title="413 — Request Too Large">
    The request body exceeds 1 MiB.

    ```json
    {
      "code": "request_too_large",
      "message": "request body too large"
    }
    ```
  </Accordion>

  <Accordion title="429 — Rate Limited">
    The caller has exceeded 5 requests per 10 minutes. Check the `Retry-After` response header for how many seconds to wait before retrying.

    ```json
    {
      "code": "too_many_requests",
      "message": "too many requests — please slow down"
    }
    ```
  </Accordion>

  <Accordion title="500 — Internal Server Error">
    An unexpected server-side error occurred.

    ```json
    {
      "code": "internal_error",
      "message": "internal server error"
    }
    ```
  </Accordion>
</AccordionGroup>
