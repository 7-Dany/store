---
title: "Get Linked Identities"
description: "Returns all OAuth providers connected to the authenticated user's account."
api: "GET http://localhost:8080/api/v1/me/identities"
authMethod: "bearer"
---

## Behaviour

**Credential fields omitted** — Access tokens and refresh tokens stored at the provider level are never included in the response. Only display-safe fields are returned.

**Empty array for no identities** — If the user has no linked OAuth providers the response is `200` with an empty `identities` array, not a `404`.

**Provider values** — Each `provider` field is a short lowercase string identifying the OAuth source. Current values: `"google"`, `"telegram"`.

## Rate Limiting

**20 requests per 1 minute per IP address**.

Clients that exceed this limit will receive a `429 Too Many Requests` response with a `Retry-After` header indicating when they may retry.

## Responses

<AccordionGroup>
  <Accordion title="200 — OK">
    The full list of linked OAuth identities. The `identities` array is empty when no providers are connected.

    | Field | Type | Notes |
    |---|---|---|
    | `provider` | string | OAuth provider name — `"google"` or `"telegram"` |
    | `provider_uid` | string | Provider-assigned user ID |
    | `provider_email` | string \| null | Email from the provider, if available. Telegram never provides an email |
    | `display_name` | string \| null | Display name stored at link time from the provider |
    | `avatar_url` | string \| null | Profile photo URL from the provider at link time |
    | `created_at` | string | ISO 8601 timestamp of when this identity was linked |

    ```json
    {
      "identities": [
        {
          "provider": "google",
          "provider_uid": "118302948571234567890",
          "provider_email": "alex@gmail.com",
          "display_name": "Alex Smith",
          "avatar_url": "https://lh3.googleusercontent.com/a/...",
          "created_at": "2024-05-01T12:00:00Z"
        },
        {
          "provider": "telegram",
          "provider_uid": "123456789",
          "provider_email": null,
          "display_name": "Alex Smith",
          "avatar_url": "https://t.me/i/userpic/320/alexsmith.jpg",
          "created_at": "2024-06-15T09:30:00Z"
        }
      ]
    }
    ```
  </Accordion>

  <Accordion title="401 — Unauthorized">
    The request did not include a valid access token, or the token has been revoked.

    **Missing token** — No `Authorization: Bearer` header was provided.

    ```json
    {
      "code": "missing_token",
      "message": "Authorization: Bearer <token> header is required"
    }
    ```

    **Invalid token** — The token is malformed, has an invalid signature, or has expired.

    ```json
    {
      "code": "invalid_token",
      "message": "invalid or expired access token"
    }
    ```

    **Token revoked** — The token has been explicitly invalidated.

    ```json
    {
      "code": "token_revoked",
      "message": "access token has been revoked"
    }
    ```
  </Accordion>

  <Accordion title="429 — Rate Limited">
    The IP address has exceeded 20 requests per minute. Check the `Retry-After` response header for how many seconds to wait before retrying.

    ```json
    {
      "code": "too_many_requests",
      "message": "too many requests — please slow down"
    }
    ```
  </Accordion>
</AccordionGroup>
