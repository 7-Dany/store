---
title: "Get Profile"
description: "Returns the authenticated user's public profile."
api: "GET http://localhost:8080/api/v1/profile/me"
---

## Behaviour

**`username`** ‚Äî Omitted from the response when the user has not yet set a username. Set a username via [`PATCH /me/username`](/api-reference/profile/username/update-username).

**`is_locked` vs admin lock** ‚Äî `is_locked` reflects a self-service OTP brute-force lock that can be cleared via [`POST /request-unlock`](/api-reference/auth/unlock/request-unlock). A separate admin-initiated lock exists but is intentionally not surfaced here ‚Äî affected users must contact support.

**`avatar_url` and `last_login_at`** ‚Äî Both fields are omitted from the response when they have not been set.

## Rate Limiting

**10 requests per 1 minute per IP address**.

Clients that exceed this limit will receive a `429 Too Many Requests` response with a `Retry-After` header indicating when they may retry.

## Responses

<AccordionGroup>
  <Accordion title="200 ‚Äî OK">
    The user's public profile. `username`, `avatar_url`, and `last_login_at` are omitted when not set.

    ```json
    {
      "id": "018f42c3-1a2b-7e4d-9f56-0a1b2c3d4e5f",
      "email": "alice@example.com",
      "display_name": "Alice",
      "username": "alice_codes",
      "avatar_url": "https://cdn.example.com/avatars/alice.png",
      "email_verified": true,
      "is_active": true,
      "is_locked": false,
      "last_login_at": "2026-03-06T10:00:00Z",
      "created_at": "2025-10-01T08:00:00Z"
    }
    ```

    | Field | Type | Notes |
    |---|---|---|
    | `id` | UUID string | Canonical UUID of the user |
    | `email` | string | Normalised email address |
    | `display_name` | string | Profile display name |
    | `username` | string | Unique handle; omitted if not yet set |
    | `avatar_url` | string | Profile picture URL; omitted if not set |
    | `email_verified` | bool | Whether the email address has been verified |
    | `is_active` | bool | Whether the account is active |
    | `is_locked` | bool | Whether the account is OTP-brute-force locked |
    | `last_login_at` | RFC 3339 timestamp | Most recent login time; omitted if never logged in |
    | `created_at` | RFC 3339 timestamp | Account creation time |
  </Accordion>

  <Accordion title="401 ‚Äî Unauthorized">
    The request is missing a valid JWT or the user ID could not be extracted from the token context.

    ```json
    {
      "code": "unauthorized",
      "message": "missing user id in context"
    }
    ```
  </Accordion>

  <Accordion title="404 ‚Äî Not Found">
    No user record was found for the ID embedded in the access token. This can occur if the account was deleted after the token was issued.

    ```json
    {
      "code": "not_found",
      "message": "user not found"
    }
    ```
  </Accordion>

  <Accordion title="429 ‚Äî Rate Limited">
    The IP address has exceeded 10 requests per 1 minute. Check the `Retry-After` response header for how many seconds to wait before retrying.

    ```json
    {
      "code": "too_many_requests",
      "message": "too many requests ‚Äî please slow down"
    }
    ```
  </Accordion>

  <Accordion title="500 ‚Äî Internal Server Error">
    An unexpected error occurred while fetching the profile.

    ```json
    {
      "code": "internal_error",
      "message": "internal server error"
    }
    ```
  </Accordion>
</AccordionGroup>
