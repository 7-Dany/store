---
title: "Revoke Session"
description: "Revokes a specific active session belonging to the authenticated user."
api: "DELETE http://localhost:8080/api/v1/profile/sessions/{id}"
---

<ParamField path="id" type="string" required>
  The UUID of the session to revoke. Must be a valid UUID v4 string. Session IDs are returned by [`GET /profile/sessions`](/api-reference/profile/session/sessions).
</ParamField>

## Behaviour

**Ownership check** — The session must belong to the authenticated user. If the session ID is valid but owned by a different user, the endpoint returns `404 not_found`, indistinguishable from a session that does not exist. This prevents cross-user session probing.

**Revoking the current session** — You can revoke your own current session. After revocation, the associated refresh token and access token are invalidated, effectively logging out that device.

## Rate Limiting

**3 requests per 15 minutes per IP address**.

Clients that exceed this limit will receive a `429 Too Many Requests` response with a `Retry-After` header indicating when they may retry.

<RequestExample>
```json Example
// No request body — the session ID is passed as a path parameter.
// DELETE /api/v1/profile/sessions/018f42c3-dead-beef-9f56-0a1b2c3d4e5f
```
</RequestExample>

<ResponseExample>
```json 204
// 204 No Content — empty body
```

```json 404
{
  "code": "not_found",
  "message": "session not found"
}
```

```json 422

  "code": "validation_error",
  "message": "session id is not a valid UUID"

```

```json 429
{

}
```
</ResponseExample>

## Responses

<AccordionGroup>
  <Accordion title="204 — No Content">
    The session was successfully revoked. The response body is empty.
  </Accordion>

  <Accordion title="401 — Unauthorized">
    The request is missing a valid JWT or the user ID could not be extracted from the token context.

    ```json
    {
      "code": "unauthorized",
      "message": "missing user id in context"
    }
    ```
  </Accordion>

  <Accordion title="404 — Not Found">
    The session does not exist or belongs to a different user. Both cases return an identical response to prevent cross-user session probing.

    ```json
    {
      "code": "not_found",
      "message": "session not found"
    }
    ```
  </Accordion>

  <Accordion title="422 — Validation Error">
    The `id` path parameter is missing or is not a valid UUID.

    ```json
    {
      "code": "validation_error",
      "message": "session id is not a valid UUID"
    }
    ```
  </Accordion>

  <Accordion title="429 — Rate Limited">
    The IP address has exceeded 3 requests per 15 minutes. Check the `Retry-After` response header for how many seconds to wait before retrying.

    ```json
    {
      "code": "too_many_requests",
      "message": "too many requests — please slow down"
    }
    ```
  </Accordion>

  <Accordion title="500 — Internal Server Error">
    An unexpected error occurred while revoking the session.

    ```json
    {
      "code": "internal_error",
      "message": "internal server error"
    }
    ```
  </Accordion>
</AccordionGroup>
