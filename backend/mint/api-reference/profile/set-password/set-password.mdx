---
title: "Set Password"
description: "Adds a password to an OAuth-only account that was created without one."
api: "POST http://localhost:8080/api/v1/profile/set-password"
authMethod: "bearer"
---

<ParamField body="new_password" type="string" required>
  The password to set on the account.

  <Expandable title="requirements">
    - 8–72 characters
    - At least one uppercase letter
    - At least one lowercase letter
    - At least one digit
    - At least one symbol
  </Expandable>
</ParamField>

## Behaviour

**OAuth-only accounts** — This endpoint is exclusively for accounts created via an OAuth provider (e.g. Google, Telegram) that have no password set. Once a password is added, the account can authenticate via both OAuth and the standard email + password login flow.

**Account already has a password** — If the account was registered with a password, or a password has already been set via this endpoint, the request is rejected with `422 password_already_set`. Use [`POST /change-password`](/api-reference/auth/password/change-password) to update an existing password.

## Rate Limiting

**5 requests per 15 minutes per authenticated user**.

This limit is keyed on the user ID from the access token, not the IP address. Clients that exceed this limit will receive a `429 Too Many Requests` response with a `Retry-After` header indicating when they may retry.

## Responses

<AccordionGroup>
  <Accordion title="200 — OK">
    The password was set successfully. The account can now authenticate via both OAuth and the email + password login flow.

    ```json
    {
      "message": "password set successfully"
    }
    ```
  </Accordion>

  <Accordion title="400 — Bad Request">
    The request body could not be parsed as valid JSON.

    ```json
    {
      "code": "bad_request",
      "message": "malformed JSON body"
    }
    ```
  </Accordion>

  <Accordion title="401 — Unauthorized">
    The request is missing a valid JWT or the user ID could not be extracted from the token context.

    ```json
    {
      "code": "unauthorized",
      "message": "missing user id in context"
    }
    ```
  </Accordion>

  <Accordion title="404 — Not Found">
    No user record was found for the ID embedded in the access token. This can occur if the account was deleted after the token was issued.

    ```json
    {
      "code": "not_found",
      "message": "user not found"
    }
    ```
  </Accordion>

  <Accordion title="413 — Payload Too Large">
    The request body exceeds the allowed limit.

    ```json
    {
      "message": "request body too large"
    }
    ```
  </Accordion>

  <Accordion title="422 — Validation Error">
    Returned for two distinct cases — check the `code` field for details.

    **Password already set** — The account already has a password. Use `POST /change-password` to update it.

    ```json
    {
      "code": "password_already_set",
      "message": "account already has a password — use change-password to update it"
    }
    ```

    **Weak or missing password** — `new_password` was empty or did not meet the strength requirements.

    ```json
    {
      "code": "validation_error",
      "message": "password must be at least 8 characters"
    }
    ```
  </Accordion>

  <Accordion title="429 — Rate Limited">
    The authenticated user has exceeded 5 requests per 15 minutes. Check the `Retry-After` response header — the value will be `180` seconds.

    ```json
    {
      "code": "too_many_requests",
      "message": "too many requests — please slow down"
    }
    ```
  </Accordion>

  <Accordion title="500 — Internal Server Error">
    An unexpected error occurred while setting the password.

    ```json
    {
      "code": "internal_error",
      "message": "internal server error"
    }
    ```
  </Accordion>
</AccordionGroup>
