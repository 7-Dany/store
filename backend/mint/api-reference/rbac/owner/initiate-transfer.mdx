---
title: "Initiate Transfer"
authMethod: "bearer"
description: "Starts an ownership transfer by generating a one-time token and emailing the target user an invitation. Only the current owner may call this endpoint. Only one pending transfer is allowed at a time."
api: "POST http://localhost:8080/api/v1/owner/transfer"
---

<ParamField body="target_user_id" type="string" required>
  The UUID of the user to whom ownership will be transferred. Must be an active, email-verified user who does not already hold the owner role.
</ParamField>

## Behaviour

**Owner-only** — The caller must hold the active owner role. Any authenticated user without this role receives `403`.

**One pending transfer at a time** — If an unexpired, unconsumed transfer token already exists, the endpoint returns `409 transfer_already_pending`. Cancel the existing transfer with [`DELETE /owner/transfer`](/api-reference/rbac/owner/cancel-transfer) before initiating a new one.

**Email invitation** — On success, a one-time transfer token is generated and an invitation email is dispatched to the target user's address. The raw token is not exposed in the HTTP response; the target user must follow the emailed link to accept.

**Target eligibility** — The target user must be active, email-verified, not already the owner, and not the same user as the caller.

**Token expiry** — The generated token has a fixed expiry (visible in `expires_at`). If the target does not accept before it expires, the transfer lapses and must be re-initiated.

## Rate Limiting

**3 requests per 24 hours per authenticated user**.

Clients that exceed this limit will receive a `429 Too Many Requests` response with a `Retry-After` header indicating when they may retry.

## Responses

<AccordionGroup>
  <Accordion title="201 — Created">
    The transfer was initiated. An invitation email has been dispatched to the target user. The response includes the transfer record ID, the target's user ID, and when the token expires.

    ```json
    {
      "transfer_id": "019a2b3c-4d5e-6f7a-8b9c-0d1e2f3a4b5c",
      "target_user_id": "018f1a2b-3c4d-7e5f-8a9b-0c1d2e3f4a5b",
      "expires_at": "2026-03-15T17:52:14.122Z"
    }
    ```
  </Accordion>

  <Accordion title="400 — Bad Request">
    The request body is not valid JSON. Fix the JSON before retrying.

    ```json
    {
      "code": "bad_request",
      "message": "malformed JSON body"
    }
    ```
  </Accordion>

  <Accordion title="401 — Unauthorized">
    No valid `Authorization: Bearer` access token was supplied. Obtain a token via [`POST /auth/login`](/api-reference/auth/login/login) and include it in the request header.

    ```json
    {
      "code": "unauthorized",
      "message": "missing or invalid access token"
    }
    ```
  </Accordion>

  <Accordion title="403 — Forbidden">
    The authenticated user does not hold the owner role.

    ```json
    {
      "code": "forbidden",
      "message": "owner role required"
    }
    ```
  </Accordion>

  <Accordion title="404 — Not Found">
    No user exists for the supplied `target_user_id`.

    ```json
    {
      "code": "user_not_found",
      "message": "target user not found"
    }
    ```
  </Accordion>

  <Accordion title="409 — Conflict">
    Returned for three distinct scenarios. Check the `code` field to distinguish them.

    **Transfer already pending** — An unexpired transfer token already exists. Cancel it with [`DELETE /owner/transfer`](/api-reference/rbac/owner/cancel-transfer) first:

    ```json
    {
      "code": "transfer_already_pending",
      "message": "an ownership transfer is already pending"
    }
    ```

    **Target is already owner** — The target user already holds the owner role:

    ```json
    {
      "code": "user_is_already_owner",
      "message": "target user is already the owner"
    }
    ```

    **Cannot transfer to self** — The `target_user_id` matches the calling owner's own ID:

    ```json
    {
      "code": "cannot_transfer_to_self",
      "message": "cannot transfer ownership to yourself"
    }
    ```
  </Accordion>

  <Accordion title="413 — Request Too Large">
    The request body exceeds the maximum allowed size.

    ```json
    {
      "code": "request_too_large",
      "message": "request body too large"
    }
    ```
  </Accordion>

  <Accordion title="422 — Validation Error">
    Returned for distinct scenarios. Check the `code` field to distinguish them.

    **Missing target** — `target_user_id` was absent or blank:

    ```json
    {
      "code": "validation_error",
      "message": "target_user_id is required"
    }
    ```

    **Target not active** — The target user's account is not active:

    ```json
    {
      "code": "user_not_active",
      "message": "target user account is not active"
    }
    ```

    **Target email not verified** — The target user has not completed email verification:

    ```json
    {
      "code": "email_not_verified",
      "message": "target user email is not verified"
    }
    ```
  </Accordion>

  <Accordion title="429 — Rate Limited">
    The authenticated user has exceeded 3 requests per 24 hours. Check the `Retry-After` response header for how many seconds to wait before retrying.

    ```json
    {
      "code": "too_many_requests",
      "message": "too many requests — please slow down"
    }
    ```
  </Accordion>

  <Accordion title="500 — Internal Server Error">
    An unexpected server-side error occurred. Contact your system administrator if this persists.

    ```json
    {
      "code": "internal_error",
      "message": "internal server error"
    }
    ```
  </Accordion>
</AccordionGroup>
