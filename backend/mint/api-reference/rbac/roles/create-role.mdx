---
title: "Create Role"
description: "Creates a new custom role."
api: "POST /rbac/roles"
---

<ParamField body="name" type="string" required>
  The display name for the role. Must be unique and descriptive.

  <Expandable title="requirements">
    - Required — cannot be blank
    - Maximum 100 characters
  </Expandable>
</ParamField>

<ParamField body="description" type="string">
  A short explanation of what this role is for. Helps other admins understand the role's purpose at a glance.
</ParamField>

<Callout icon="key" color="#FFC107">
  Requires the **`rbac:manage`** permission.
</Callout>

## Behaviour

Creates an active, non-system role. Newly created roles have no permissions — use [`POST /rbac/roles/{id}/permissions`](/api-reference/rbac/roles/add-role-permission) to assign permissions after creation.

<RequestExample>
```json Example
{
  "name": "Store Manager",
  "description": "Can manage products and orders."
}
```
</RequestExample>

<ResponseExample>
```json 201
{
  "id": "018e4c3a-1b2d-7f4e-9c0a-2d3e4f5a6b7c",
  "name": "Store Manager",
  "description": "Can manage products and orders.",
  "is_system_role": false,
  "is_owner_role": false,
  "is_active": true,
  "created_at": "2024-06-01T10:00:00Z"
}
```

```json 422
{
  "code": "validation_error",
  "message": "name is required"
}
```
</ResponseExample>
