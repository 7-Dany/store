---
title: "Update Role"
description: "Updates the name or description of a custom role."
api: "PATCH http://localhost:8080/admin/rbac/roles/{id}"
---

<ParamField path="id" type="string" required>
  The UUID of the role to update.
</ParamField>

<ParamField body="name" type="string">
  New display name for the role.

  <Expandable title="requirements">
    - Cannot be blank if provided
    - Maximum 100 characters
  </Expandable>
</ParamField>

<ParamField body="description" type="string">
  New description for the role. Pass an empty string to clear it.
</ParamField>

<Callout icon="key" color="#FFC107">
  Requires the **`rbac:manage`** permission.
</Callout>

## Behaviour

This is a partial update — only the fields you include are changed. At least one field (`name` or `description`) must be present in the request body.

System roles and the owner role cannot be modified. Attempting to update them returns `409 Conflict`.

<RequestExample>
```json Example
{
  "name": "Senior Store Manager",
  "description": "Manages products, orders, and staff."
}
```
</RequestExample>

<ResponseExample>
```json 200
{
  "id": "018e4c3a-1b2d-7f4e-9c0a-2d3e4f5a6b7c",
  "name": "Senior Store Manager",
  "description": "Manages products, orders, and staff.",
  "is_system_role": false,
  "is_owner_role": false,
  "is_active": true,
  "created_at": "2024-06-01T10:00:00Z"
}
```

```json 409
{
  "code": "system_role_immutable",
  "message": "system roles cannot be modified"
}
```

```json 422
{
  "code": "validation_error",
  "message": "at least one field (name or description) must be provided"
}
```
</ResponseExample>

## Responses

<AccordionGroup>

    ```json
    {

      "name": "Senior Store Manager",

      "is_system_role": false,
      "is_owner_role": false,

    ```
  </Accordion>

    ```json
    {
      "code": "unauthorized",

    }
    ```
  </Accordion>

  <Accordion title="403 — Forbidden">
    The authenticated user does not hold the `rbac:manage` permission.

    ```json
    {

      "message": "forbidden"
    }
    ```
  </Accordion>

    ```json

    ```
  </Accordion>

  </Accordion>

  <Accordion title="422 — Validation Error">

    ```json
      "message": "at least one field (name or description) must be provided"

    ```
  </Accordion>

    ```json
    {
      "code": "internal_error",
      "message": "internal server error"
    }
    ```
  </Accordion>
</AccordionGroup>
