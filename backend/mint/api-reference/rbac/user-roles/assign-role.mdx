---
title: "Assign User Role"
description: "Assigns or replaces the role for a given user."
api: "PUT http://localhost:8080/admin/rbac/users/{user_id}/role"
---

<ParamField path="user_id" type="string" required>
  The UUID of the user to assign a role to.
</ParamField>

<ParamField body="role_id" type="string" required>
  The UUID of the role to assign. Must correspond to an existing active role.
</ParamField>

<ParamField body="granted_reason" type="string" required>
  A business justification for the assignment. Required for accountability.

  <Expandable title="requirements">
    - Cannot be empty or whitespace
    - Maximum 500 characters
  </Expandable>
</ParamField>

<ParamField body="expires_at" type="string">
  Optional expiry timestamp in ISO 8601 format. When omitted the grant is permanent. When provided, the role is treated as temporary and will no longer be active after this time.

  <Expandable title="requirements">
    - Must be at least 5 minutes in the future
    - Cannot exceed 90 days from the time of the request
  </Expandable>
</ParamField>

<Callout icon="key" color="#FFC107">
  Requires the **`rbac:manage`** permission.
</Callout>

## Behaviour

Assigns a role to the target user, replacing any existing non-owner role assignment in a single operation. Each user can hold exactly one role at a time — this is enforced at the database level.

**Self-assignment is not allowed.** You cannot modify your own role assignment.

**The owner role cannot be assigned through this endpoint.** If the target user already holds the owner role, the request is rejected. The owner role is only assignable via the [`POST /admin/rbac/owner/bootstrap`](/api-reference/rbac/bootstrap/bootstrap) endpoint.

Temporary grants must specify an `expires_at` value. Once expired, the user is treated as having no role until a new assignment is made. The expiry window is enforced server-side: a minimum of 5 minutes and a maximum of 90 days from now.

<RequestExample>
```json Permanent grant
{
  "role_id": "029f5d4b-2c3e-8a5f-0d1b-3e4f5a6b7c8d",
  "granted_reason": "Promoted to store manager after onboarding."
}
```

```json Temporary grant
{
  "role_id": "029f5d4b-2c3e-8a5f-0d1b-3e4f5a6b7c8d",
  "granted_reason": "Temporary access while primary manager is on leave.",
  "expires_at": "2024-07-01T10:00:00Z"
}
```
</RequestExample>

<ResponseExample>
```json 200
{
  "user_id": "018e4c3a-1b2d-7f4e-9c0a-2d3e4f5a6b7c",

  "role_name": "Store Manager",
  "is_owner_role": false,
  "granted_reason": "Promoted to store manager after onboarding.",
```

## Responses

<AccordionGroup>
  <Accordion title="200 — OK">
    The role was successfully assigned or replaced. Returns the full updated role assignment. `expires_at` is `null` for permanent grants.

    ```json
    {
      "user_id": "018e4c3a-1b2d-7f4e-9c0a-2d3e4f5a6b7c",
      "role_id": "029f5d4b-2c3e-8a5f-0d1b-3e4f5a6b7c8d",
      "role_name": "Store Manager",
      "is_owner_role": false,
      "granted_reason": "Promoted to store manager after onboarding.",
      "granted_at": "2024-06-01T10:00:00Z",
      "expires_at": null
    }
    ```
  </Accordion>

  <Accordion title="401 — Unauthorized">
    The request is missing a valid access token.

    ```json
    {
      "code": "unauthorized",
      "message": "missing or invalid access token"
    }
    ```
  </Accordion>

  <Accordion title="403 — Forbidden">
    The authenticated user does not hold the `rbac:manage` permission.

    ```json
    {
      "code": "forbidden",
      "message": "forbidden"
    }
    ```
  </Accordion>

  <Accordion title="409 — Conflict">
    Returned in two cases: attempting to modify your own role assignment, or attempting to reassign a user who already holds the owner role.

    ```json
    {
      "code": "cannot_modify_own_role",
      "message": "you cannot modify your own role assignment"
    }
    ```

    ```json
    {
      "code": "cannot_reassign_owner",
      "message": "owner role cannot be reassigned via this route"
    }
    ```
  </Accordion>

  <Accordion title="422 — Validation Error">
    One or more required fields are missing or invalid. Also returned when `role_id` does not correspond to an active role.

    ```json
    {
      "code": "role_id_required",
      "message": "role_id is required"
    }
    ```

    ```json
    {
      "code": "granted_reason_required",
      "message": "granted_reason is required"
    }
    ```

    ```json
    {
      "code": "role_not_found",
      "message": "role not found"
    }
    ```
  </Accordion>

  <Accordion title="500 — Internal Server Error">
    An unexpected error occurred on the server.

    ```json
    {
      "code": "internal_error",
      "message": "internal server error"
    }
    ```
  </Accordion>
</AccordionGroup>
