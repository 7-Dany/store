Resetting a forgotten password takes three API calls and three UI screens. This guide walks through the complete flow, including how the grant token works and which errors need to send the user back to an earlier screen.

---

## Overview

```
POST /auth/password/reset          →  OTP sent to email (always 202)
POST /auth/password/reset/verify   →  email + code  →  { reset_token }
PUT  /auth/password/reset          →  reset_token + new_password  →  200
POST /auth/login                   →  sign in with new password
```

---

## Screen 1 — Enter email

Send the user's email to [`POST /auth/password/reset`](/api-reference/auth/password/forgot-password):

```json
{
  "email": "alice@example.com"
}
```

The response is always `202 Accepted` with the same body regardless of whether the email is registered:

```json
{
  "message": "if that email is registered and verified, a password reset code has been sent"
}
```

Show this message as-is. Do not confirm or deny whether the account exists.

**Cooldown** — If the user submits again within 60 seconds, no new code is sent but the response is still `202`. Tell the user to check their inbox before requesting again.

---

## Screen 2 — Enter the code

The user checks their email and enters the 6-digit code. Submit it along with the email to [`POST /auth/password/reset/verify`](/api-reference/auth/password/verify-reset-code):

```json
{
  "email": "alice@example.com",
  "code": "847291"
}
```

On success you receive a grant token:

```json
{
  "reset_token": "3f2a1b4c-...",
  "expires_in": 600
}
```

Store the `reset_token` in your frontend state and advance the user to screen 3. The token expires in 10 minutes.

**The code is not consumed here.** Verification happens without touching the token's used state — this is purely a check so you can confirm the code is correct before asking the user to choose a new password.

**If the code is wrong** — the server returns `422 validation_error`. Show an "incorrect code" message and let the user try again. The attempt counter still increments on wrong guesses, so after too many failures the code locks and returns `429`, at which point the user must go back to screen 1.

---

## Screen 3 — Enter new password

The user chooses a new password. Submit the grant token and new password to [`PUT /auth/password/reset`](/api-reference/auth/password/reset-password):

```json
{
  "reset_token": "3f2a1b4c-...",
  "new_password": "NewP@ssw0rd1!"
}
```

On success:

```json
{
  "message": "password reset successfully"
}
```

At this point every active session on every device is signed out and all previously issued access tokens are immediately invalid. Send the user to the login screen.

---

## Error handling

| Screen | Response | Cause | Action |
|---|---|---|---|
| 2 | `410 token_expired` | The OTP code has expired | Back to screen 1 |
| 2 | `422 validation_error` | Wrong code, already used, or no reset requested | Show "incorrect code", let user retry |
| 2 | `429 too_many_requests` | Too many wrong codes — code is locked | Back to screen 1 for a fresh code |
| 2 | `429 rate_limited` | IP exceeded 5 req / 10 min | Show `Retry-After` value |
| 3 | `422` — invalid/expired grant token | Token expired (10 min), already used, or not found | Back to screen 1 |
| 3 | `422` — same password | New password matches current | Stay on screen 3, ask for a different password |
| 3 | `422` — weak password | Password doesn't meet requirements | Stay on screen 3, show requirements |
| 3 | `429 rate_limited` | IP exceeded 5 req / 10 min | Show `Retry-After` value |

---

## Screen 4 — Sign in

Redirect the user to your login screen ([`POST /auth/login`](/api-reference/auth/session/login)). Their old password no longer works.

```json
{
  "identifier": "alice@example.com",
  "password": "NewP@ssw0rd1!"
}
```


    ```json Response — 202

    }
    ```
  </Step>

  <Step title="POST /auth/password/reset/verify — Validate the OTP">

    ```json Request
    {

      "code": "847291"
    }
    ```

    ```json Response — 200
    {
      "reset_token": "3f2a1b4c-...",
      "expires_in": 600
    }
    ```

